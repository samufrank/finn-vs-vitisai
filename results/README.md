# Results

Benchmark results as JSON files, auto-generated by `board/benchmark.py`.

## Naming convention

`{tool}_{arch}-{hidden_sizes}_{dataset}_b{batch_size}_{timestamp}.json`

Examples:
- `vitisai_mlp-256x256x128_mnist_b1_20260317_021234.json`
- `vitisai_mlp-256x256x128_cifar10_b64_20260317_023456.json`
- `finn_mlp-64x32_mnist_b1_20260317_031234.json`

The `--name` flag on benchmark.py controls the `{tool}_{arch}-{hidden_sizes}` portion.

## JSON structure

Each file contains:
- `config`: model path, dataset, batch size, DPU input/output shapes, board info, timestamp
- `idle_power`: mean and std of idle power baseline
- `runs`: array of per-run results (accuracy, throughput, power, energy, latency)
- `summary`: aggregated mean and std across all runs

## Current data

### vitis_ai/
MLP (784->256->256->128->10) batch size ablation on both MNIST and CIFAR-10.
Batch sizes: 1, 4, 8, 16, 32, 64. Each run includes 5 measured trials.
Note: this model does not fit on FINN at INT8 (BRAM overflow). These results
characterize Vitis AI batch scaling behavior but cannot be directly compared
to FINN. A matched comparison requires using a smaller model that fits both tools.

### finn/
Empty — FINN models have been compiled but not yet deployed and benchmarked.
