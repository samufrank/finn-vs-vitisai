# Results

Benchmark results as JSON files, generated by `board/benchmark.py` or `board/vta_infer.c`, then merged with power data by `board/merge_power.py`.

## Naming Convention

`{toolchain}_{model}_{dataset}_{precision}[_{clock}][_{runtime}].json`

Examples:
- `finn_mlp-64x32_mnist_int8.json`
- `vta_mlp-64x32_mnist_int8_250mhz_c.json`
- `dpu_mlp-64x32_mnist_int8_300mhz.json`
- `finn_cnn-8x16_cifar10_int8.json`

Clock is included for VTA and DPU (user-controlled). FINN auto-selects clock.
Runtime suffix (`_c`) is added when non-default (e.g., VTA C runner vs Python).

Power CSVs use the same base name with `_power.csv`. Timeline plots use `_power.png`.

## JSON Structure

Each file contains:
- `config`: model path, dataset, toolchain, board info, timestamps
- `idle_power`: mean and std of idle power baseline (from FNB58 merge)
- `runs`: array of per-run results (accuracy, throughput, power, energy, latency)
- `summary`: aggregated mean and std across all runs

Power fields are present only in merged results (after `merge_power.py`). Raw benchmark JSONs from the board contain timing and accuracy only.

## Current Data

All results from AUP-ZU3 (ZU3EG) with FNB58 external power measurement.

### finn/
- MLP tiny [64,32] MNIST — INT8, Brevitas QAT
- CNN tiny [8,16] MNIST — INT8, Brevitas QAT
- CNN tiny [8,16] CIFAR-10 — INT8, Brevitas QAT (model too small for task, ~10% accuracy)

### vta/
- MLP tiny [64,32] MNIST — INT8, 250 MHz, Python and C runners
- MLP tiny [64,32] MNIST — INT4, 200 MHz, Python runner (Brevitas QAT w4a4, 93.08%, 245.6 FPS, 18.13 mJ)
- MLP tiny [64,32] MNIST — INT4, 200 MHz, C runner (93.08%, 1266 FPS, 3.15 mJ)
- CNN tiny [8,16] MNIST — INT8, 250 MHz, Python runner
- CNN tiny [8,16] MNIST — INT4, 166 MHz, Python runner, INT4-o8 bitstream (per-channel BN-fold, zero-point activation offset, 81.57%, 29.2 FPS, 142.0 mJ)

Note on CNN INT4: requires a mixed-precision bitstream (int4 input/weights, int8 DMA output) because VTA's per-tensor shift-right with 4-bit output width cannot preserve inter-layer signal through BN-amplified channels. Zero-point offset maps Brevitas unsigned [0,15] activations to VTA signed [-8,7] with bias correction, recovering the full activation range. Timing closed at 166 MHz (down from 200 MHz for pure INT4, 250 MHz for INT8).

### vitis_ai/
- MLP tiny [64,32] MNIST — INT8, B512 DPU, 300/600 MHz
- CNN tiny [8,16] MNIST — INT8, B512 DPU, 300/600 MHz

### finn-t/
- Trained RadioML 2018 transformer — INT4, 3 heads, emb_dim=96, 1 layer, 122k params, Brevitas QAT. 72.12% accuracy (eval, SNR ≥ -6 dB), 1460.8 FPS (C runner, double-buffered), 2.758 mJ/inference, 0.411 W dynamic power. First known trained-transformer deployment on ZU3EG outside Paderborn group.
- DummyTransformer Config A (2h, D=32, T=16, 2-bit) — resource estimation and bitstream archive (random weights, no accuracy metric)
